## 4. 전체 액티비티 다이어그램 (Overall Activity Diagram)

### 시스템 흐름 요약

1.  **회원가입/로그인**: 고객은 시스템 이용을 위해 로그인하거나 회원가입(Customer-uc1, uc2)을 수행한다.
2.  **주문 생성**:
    *   고객은 메뉴를 선택하고 옵션을 설정한다 (**메뉴 선택, Customer-uc5**).
    *   또는 음성을 통해 주문을 생성할 수 있다 (**음성 주문, Customer-uc6**).
    *   최종적으로 주문을 확정하면 시스템이 재고를 확인하고 주문을 저장한다 (**주문 확정, Customer-uc3**). 이때 별도의 외부 결제 시스템 연동 없이 주문이 승인된다.
3.  **주문 처리 (주방)**:
    *   주방 직원은 접수된 주문을 확인하고 조리를 시작한다 (**주문 처리, Staff-uc1**).
    *   조리가 완료되면 주문 상태를 ‘준비됨(Ready)’으로 변경한다.
4.  **배달 처리 (배달)**:
    *   배달 직원은 준비된 주문을 확인하고 고객에게 배달한다 (**배달 처리, Staff-uc2**).
    *   배달이 완료되면 주문 상태를 ‘배달 완료(Delivered)’로 변경한다.
5.  **재고 관리**:
    *   직원은 수시로 재고 상태를 확인하고 부족한 물품을 확충한다 (**재고 관리, Staff-uc3**).

### 상태 전이 요약

*   **Pending (접수됨)**: 주문 확정 직후의 상태.
*   **In Progress (조리 중)**: 주방 직원이 확인하여 조리가 진행 중인 상태.
*   **Ready (준비됨)**: 조리가 완료되어 배달을 기다리는 상태.
*   **Delivered (배달 완료)**: 고객에게 전달이 완료된 최종 상태.
